
# 앱인토스 출시 체크리스트 (Apps-In-Toss Checklist)
출처: https://developers-apps-in-toss.toss.im/checklist/app-game.html

이 문서는 "룸메이트체크(RoommateCheck)"를 앱인토스로 출시하기 위한 실행 체크리스트입니다.
원래 게임 출시 가이드이지만, 대부분의 UI/UX 요구사항은 인터랙티브 미니앱에도 동일하게 적용됩니다.

## 1. UI 및 네비게이션 필수 요건
- [ ] **풀스크린 모드 (Full Screen)**: 앱은 반드시 전체 화면을 채워야 합니다.
  - 레터박스(검은 여백)가 생기지 않도록 해야 합니다.
  - 세이프 에어리어(노치, 다이내믹 아일랜드)를 침범하지 않게 여백을 처리해야 합니다.
  - 스크롤 오류나 클릭 불가능한 영역이 없어야 합니다.
  - 핀치 줌(손가락으로 확대) 방지를 위해 `meta viewport` 설정을 점검하세요: `initial-scale=1, maximum-scale=1, user-scalable=no`.

- [ ] **네비게이션 바 (Navigation Bar)**:
  - 토스에서 제공하는 네비게이션 바 스타일과 일관성을 유지해야 합니다.
  - 우측 상단에는 반드시 `...` (더보기) 및 `X` (닫기) 버튼이 있어야 합니다.
  - `X` 버튼이 앱 내부의 중요한 버튼과 겹치지 않도록 배치해야 합니다.
  - **종료 확인 모달**: 앱을 닫을 때 반드시 종료 확인 모달을 띄워야 합니다:
    - 문구: "$서비스명$을 종료할까요?" (예: 룸메이트체크를 종료할까요?)
    - 버튼: "취소" / "종료하기" (브랜드 포인트 컬러 적용)

- [ ] **테마 (Theme)**:
  - **라이트 모드 필수**: 미니앱은 현재 다크 모드를 지원하지 않으므로, 무조건 **라이트 모드**로 동작해야 합니다.
  - 룸메이트체크의 "토스 모드(Toss Mode)"는 이미 화이트/블루 테마이므로 이 기준을 충족합니다.

## 2. 인터랙션 및 UX
- [ ] **제스처 (Gestures)**:
  - 꼭 필요한 경우(예: 상세 이미지 보기)가 아니면 핀치 줌 기능을 막아야 합니다.
- [ ] **사운드 (Sound)**:
  - 배경음이나 효과음이 있다면, 앱이 백그라운드로 내려갈 때 소리가 즉시 멈춰야 합니다.

## 3. 로그인 및 권한
- [ ] **토스 로그인 (Toss Login)**:
  - 사용자가 로그인(약관 동의)을 하기 전에 이 서비스가 어떤 가치를 제공하는지 명확히 이해할 수 있어야 합니다.
  - **진입 흐름**: 인트로 페이지(서비스 소개) -> 로그인 요청(약관 동의) -> 앱 메인 진입 순서여야 합니다.
  - **연동 해제**: 사용자가 토스 앱 설정에서 "로그인 연동 해제"를 한 경우, 재진입 시 기존 데이터가 남지 않고 새로운 유저로 인식해야 합니다.

## 4. 정책 및 콘텐츠
- [ ] **프로모션 (Promotions)**: 현금성 보상(현금, 상품권 등) 지급은 엄격히 금지됩니다. 포인트 지급 시에는 반드시 '토스 포인트' 전환에 대한 승인 절차를 거쳐야 합니다.
- [ ] **연령 등급**: 콘솔 설정에 따라 연령 등급 표시가 필요할 수 있습니다.

## 5. 기술적 검증
- [ ] **데이터 사용량**: 빠른 로딩을 위해 이미지 등 에셋 용량을 최적화해야 합니다.
- [ ] **권한 요청**: 서비스에 꼭 필요한 최소한의 권한만 요청해야 합니다.
