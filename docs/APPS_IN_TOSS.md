This document outlines the resources for deploying "RoommateCheck" as an "App-In-Toss".

## 1. What is App-In-Toss?
Deploy web-based services directly inside the Toss App, reaching 30 million users without separate installation.

## 2. Key Integration Paths
### Web (WebView)
- **Guide**: [Start with Web](https://developers-apps-in-toss.toss.im/tutorials/webview.html)
- Best for existing web apps (React, Next.js).
- Need to ensure responsive design matches Toss UI guidelines.

### React Native
- **Guide**: [Start with React Native](https://developers-apps-in-toss.toss.im/tutorials/react-native.html)
- Use standard RN components adapted for the Toss environment.

## 3. Core Features for MateCheck
### Authentication & User
- **Game/App Profile**: Get user identity securely.

### Payments & Points
- **Toss Pay**: One-tap payment integration. [Guide](https://developers-apps-in-toss.toss.im/tosspay/intro.html)
- **In-App Purchase**: Sell digital goods (e.g., premium themes). [Guide](https://developers-apps-in-toss.toss.im/iap/intro.html)
- **Toss Points**: Use points for rewards or promotions. [Guide](https://developers-apps-in-toss.toss.im/promotion/intro.html)

### Graphic & Design
- **Toss Design System (TDS)**: Use "Toss App Builder" or Figma kits to match the native look. [Guide](https://developers-apps-in-toss.toss.im/design/prepare/deus.html)
